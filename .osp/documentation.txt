МОЖНО ПОТОМ ЗАПИХНУТЬ ЭТО НА ОДНУ ИЗ СТРАНИЦ САЙТА
----------------------------------------------------

1. /app
/controllers: Здесь будут файлы контроллеров, которые обрабатывают запросы от пользователя. 
Они принимают данные из URL, взаимодействуют с моделями и выбирают, какое представление отобразить. 
Например, контроллер может получать данные о пользователе и передавать их в представление для отображения.

/models: В этой папке будут модели, которые представляют данные и бизнес-логику приложения. 
Модели отвечают за взаимодействие с базой данных (например, получение, добавление или обновление данных). 
Например, модель "User " может содержать методы для работы с данными пользователей.

/views: Здесь будут файлы представлений, которые отвечают за отображение информации пользователю. 
Представления получают данные от контроллеров и формируют HTML-страницы. Например, представление 
для профиля пользователя будет отображать информацию о пользователе.

2. /public
index.php: Это главный файл, который будет точкой входа в приложение. Когда пользователь обращается к вашему сайту, именно этот файл обрабатывает запросы. Он будет выполнять начальную настройку и вызывать нужные контроллеры в зависимости от URL.
3. /config (если понадобится)
Здесь могут находиться конфигурационные файлы, например, для подключения к базе данных, настройки маршрутизации или другие параметры приложения.
Принцип работы
Когда пользователь вводит URL в браузере:

Запрос попадает в index.php.
index.php определяет, какой контроллер и метод нужно вызвать.
Контроллер обрабатывает запрос, взаимодействует с моделью для получения данных и передает их в представление.
Представление формирует HTML-страницу и отправляет её обратно пользователю.
